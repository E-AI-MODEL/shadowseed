# Transfer evaluation layer

Deze map markeert de toekomstige domeintransfer-laag.

Doel:

- testen of seedkwaliteit overeind blijft buiten de bekende benchmarkdomeinen;
- holdout-domeinen en nieuwe tekstvormen apart beoordelen;
- transfer drop en domeinvariatie zichtbaar maken.

Hoort hierbij:

- nog geen stabiel CLI-command;
- later domein-holdout batches en transfer summaries.

Werkregel:

> Deze laag start pas inhoudelijk als open-review en adversarial evaluatie stabieler zijn.

Geen gedrag wordt door deze map gewijzigd. Ze legt alleen de bedoelde eigenaar van toekomstige transferlogica vast.
